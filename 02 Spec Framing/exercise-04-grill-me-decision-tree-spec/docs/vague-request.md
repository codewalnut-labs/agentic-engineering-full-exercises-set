# Vague Entitlement Request

Product asks: "Make entitlement overrides more flexible for enterprise customers."

Known constraints:
- Sales wants account-level exceptions for renewal negotiations.
- Support needs clear messages when a customer cannot enable a feature.
- Finance needs downgrade behavior to match billing cycle boundaries.
- Security requires an audit trail for blocked and successful attempts.

Unanswered questions:
- Do child accounts inherit parent overrides?
- What wins when plan defaults and account overrides disagree?
- Can an expired override be renewed automatically?
- Should downgrade changes apply immediately or at cycle close?
