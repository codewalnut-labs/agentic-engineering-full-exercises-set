# Exercise 04 : Superpowers TDD Red-Green Refactor

## Your Mission

Your mission is to implement retry behavior through a disciplined red-green-refactor loop.

You are given a repository with unclear invoice retry rules and a tempting broad rewrite path.

The duration for this challenge is 30 min or less.

## Project

[invoice-retry-app](./invoice-retry-app) contains the invoice retry workflow for this exercise.

## How To Go About It

Use [Superpowers](https://github.com/obra/superpowers) or an equivalent TDD agent workflow.

Ask your coding agent to inspect `invoice-retry-app/`, write failing tests first, implement the behavior, refactor safely, and verify it.

## Evidence

Produce the tests, implementation change, and red-green-refactor evidence.

Raise the completed work as a PR for getting verified with our team.
