# Agentic Engineering Rubric

This curriculum follows a core agentic engineering principle: agents replace typing, not thinking. An accountable engineer or leader must set the contract, direct the agent, verify the work, and improve the system around repeated work.

## Non-Negotiable Completion Rule

No exercise is complete with only Markdown. Each exercise must produce a working change in code, tests, automation, configuration, skills, hooks, CI, or operating workflow, plus evidence that the change was verified.

## Mastery Gates

- Toolchain Setup: the agent can work safely in the repo using checked-in setup context.
- Spec Framing: the implementation starts from testable examples and bounded scope.
- Context Engineering: project knowledge lives in versioned files and queryable maps, not chat paste.
- Test Automation: tests fail for real regressions and stay stable across repeated runs.
- Skill Packaging: repeated workflows become focused skills with evals and trigger discipline.
- Multi-Agent Workflows: parallelism is isolated, reviewable, and integrated by one accountable owner.
- Docs & Diagrams: architecture docs are generated from and verified against code.
- Evidence-led PRs: reviewers see intent, checks, artifacts, risks, and rollback proof.
- Code Review: fresh review, specialist checks, human triage, and re-review protect merge quality.
- Token Economics: context, model, effort, and automation choices are explicit and measurable.
- Agentic Refactoring: behavior is characterized before cleanup and stays green throughout.
- Agentic Retrospective: session evidence becomes better rules, skills, hooks, and habits.

## Required Evidence Pattern

Every exercise solution should include:

- The working diff or config change.
- The tests, scripts, hooks, CI jobs, skills, or agent workflow used to verify it.
- The commands run with pass/fail results.
- A short decision note explaining trade-offs, residual risks, and what a reviewer should inspect first.

## Leadership Lens

For leaders, the goal is not to become the fastest prompt writer. The goal is to design an engineering system where teams can use Codex, Claude Code, Cursor, and future agents with guardrails, evidence, cost discipline, and accountable review.
