# Senior Review Checklist

- Behavior: changed logic matches the requested user outcome and preserves edge cases.
- Security: no unsafe rendering, secret exposure, overbroad access, or unvalidated privileged action.
- Accessibility: interactive elements keep roles, names, keyboard behavior, focus, and visible state.
- Performance: new loops, caching, rendering, or network calls have bounded cost.
- Tests: important findings map to tests, not only reviewer memory.
- Evidence: the PR explains intent, commands run, residual risk, and rollback where relevant.
