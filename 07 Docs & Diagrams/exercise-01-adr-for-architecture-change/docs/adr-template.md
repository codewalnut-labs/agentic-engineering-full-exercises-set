# Adr Template

This is a seeded senior-lab input for ADR for Architecture Change. It gives the learner concrete constraints to inspect, implement, test, and verify.

## Operating Context

Architecture decision for extracting workflow state

## Concrete Inputs

- state boundary
- service module
- ADR
- alternative

## Seeded Risks

- ADR is written before behavior is verified
- service extraction changes selected item persistence
- diagram omits the new service boundary

## Verification Expectations

- boundary behavior test
- ADR completeness check
- diagram verification
- architecture smoke

## Agent Workflow Constraint

The learner must use an agent to inspect and plan, but the final implementation, review, and verification remain owned by the senior engineer.
