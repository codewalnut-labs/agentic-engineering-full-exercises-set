# Exercise 01 : Structurizr ADR Architecture Change

## Your Mission

Your mission is to document an architecture decision after validating the current workflow state design.

You are given a repository where workflow state is split between local UI state and service boundaries.

The duration for this challenge is 30 min or less.

## Project

[workflow-state-app](./workflow-state-app) contains the workflow state architecture for this exercise.

## How To Go About It

Use [Structurizr](https://structurizr.com/) or Structurizr DSL to model the architecture decision.

Ask your coding agent to inspect `workflow-state-app/`, create the ADR, update the architecture view, and verify both against code.

## Evidence

Produce the ADR, architecture diagram source, and verification notes.

Raise the completed work as a PR for getting verified with our team.
