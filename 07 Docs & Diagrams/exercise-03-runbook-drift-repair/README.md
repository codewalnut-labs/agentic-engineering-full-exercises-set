# Exercise 03 : Playwright Runbook Drift Repair

## Your Mission

Your mission is to repair a stale runbook by proving the current product behavior first.

You are given a repository where support documentation no longer matches the live incident flow.

The duration for this challenge is 30 min or less.

## Project

[runbook-drift-app](./runbook-drift-app) contains the incident runbook workflow for this exercise.

## How To Go About It

Use [Playwright MCP](https://playwright.dev/docs/getting-started-mcp) or Playwright traces to capture the real workflow.

Ask your coding agent to inspect `runbook-drift-app/`, verify the flow in the browser, update the runbook, and fix one drift-causing issue.

## Evidence

Produce the repaired runbook, evidence capture, fix, and verification output.

Raise the completed work as a PR for getting verified with our team.
