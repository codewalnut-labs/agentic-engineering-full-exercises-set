# Exercise Matrix

## 01. Toolchain Setup

New sessions need almost no re-explaining, checks are discoverable, and sensitive tools or paths are guarded.

- Exercise 01: Agent Onboarding Kit - Configure a coding agent so it can understand and work on a repository like a new engineering teammate.
- Exercise 02: Agent Guardrails - Create safety guardrails for a coding agent before allowing it to work independently on a repository.

## 02. Spec Framing

The implementation starts from clear acceptance criteria, concrete examples, non-goals, and reviewable slices.

- Exercise 01: Spec Driven Feature Development - Convert a vague subscription-management request into Spec Kit artifacts before implementation.
- Exercise 02: Superpowers Feature Implementation - Use Superpowers skills to plan, implement, and verify team invitation functionality.

## 03. Context Engineering

Architecture, commands, conventions, and repeated corrections live in versioned project context, not only in chat.

- Exercise 01: Context7 Fresh Docs Rescue - Use Context7 to refresh stale package knowledge, build a tight context pack, and fix the bug with evidence.
- Exercise 02: Serena Token-Budgeted Feature Delivery - Use Serena-style semantic navigation to deliver a feature with a visible context budget.
- Exercise 03: MCP Product Rules Context Server - Build an MCP-style context server for product rules and use it to complete a focused app change.
- Exercise 04: Graphiti NFR Context Memory - Use Graphiti-style memory to make an NFR audit actionable instead of chat-only.
- Exercise 05: Graphify Knowledge Graph Lab - Create a Graphify-style codebase graph and use it to guide a cross-cutting billing change.
- Exercise 06: Intent Layer Repo Map - Create folder-level intent context and prove it by completing a safe incident workflow improvement.

## 04. Test Automation

Tests fail for real regressions, use resilient locators, run independently, and produce debugging evidence.

- Exercise 01: Playwright MCP Checkout Rescue - Use Playwright MCP or traces to stabilize flaky checkout E2E tests.
- Exercise 02: MSW Network Boundary Tests - Use MSW to add meaningful component tests around network-driven UI states.
- Exercise 03: Pact Contract Test Gate - Use Pact-style contract testing to protect a UI and rules API boundary.
- Exercise 04: Superpowers TDD Red-Green Refactor - Use a Superpowers-style TDD loop to implement retry rules without a broad rewrite.

## 05. Skill Packaging

Repeated workflows become focused, versioned skills that trigger correctly and produce consistent output.

- Exercise 01: Copilot Review Skill Pack - Package a reusable code review checklist as an agent skill and run it against the app.
- Exercise 02: Release Notes Skill Factory - Create a reusable release-notes skill and validate it against repository changes.
- Exercise 03: jscodeshift Migration Playbook Skill - Package a migration skill that uses jscodeshift and prove it on one safe slice.
- Exercise 04: Promptfoo Skill Trigger Eval Harness - Use Promptfoo to evaluate and improve skill trigger behavior.
- Exercise 05: Cross-Agent Skill Portability Pack - Turn a chat-only compliance workflow into a portable skill package.

## 06. Multi-Agent Workflows

Independent tasks run side by side in isolated branches or worktrees, with the main thread owning integration.

- Exercise 01: Parallel Worktree Feature Split - Coordinate parallel worktree slices and integrate them without overlapping edits.
- Exercise 02: Subagent NFR Swarm - Run specialist agent reviews, fix the highest-value NFR issues, and keep one integration owner.
- Exercise 03: Conflict-Tolerant Migration Board - Coordinate a batched migration with file ownership, conflict logging, and integrated verification.
- Exercise 04: Kanban Worktree Control Plane - Create an agent-ready Kanban control plane and complete one isolated card.

## 07. Docs & Diagrams

Decisions, diagrams, and non-obvious system behavior are recorded while context is fresh and verified against code.

- Exercise 01: Structurizr ADR Architecture Change - Use Structurizr to document and verify a workflow-state architecture decision.
- Exercise 02: Mermaid C4 and Sequence Diagrams - Create Mermaid C4-style and sequence diagrams from actual code paths.
- Exercise 03: Playwright Runbook Drift Repair - Use browser evidence to repair stale runbook documentation and one drift-causing issue.
- Exercise 04: Excalidraw Workflow Reconstruction - Use an Excalidraw skill to rebuild workflow diagrams from real implementation paths.
- Exercise 05: Graphiti Codebase Graph to Diagrams - Turn codebase graph context into verified diagrams and use them to guide a notification change.
- Exercise 06: Feature visualization - Generate architecture, process flow, sequence, and ER diagrams for a payment integration feature.

## 08. Evidence-led PRs

Reviewers see intent, checks, artifacts, risks, and rollback notes without reconstructing the work.

- Exercise 01: GitHub Actions PR Evidence Pack - Create a GitHub Actions evidence pack that accompanies a small app change.
- Exercise 02: OpenFeature Rollback Proof - Use OpenFeature-style flags to make a feature reviewable and reversible.
- Exercise 03: Lighthouse CI Evidence Gate - Use Lighthouse CI evidence to prove performance and accessibility quality on a UI change.

## 09. Code Review

Clean-looking code is checked for behavior, security, accessibility, performance, tests, and maintainability.

- Exercise 01: Semgrep Security Review Gauntlet - Use Semgrep and manual review to catch real blockers in an agent-written diff.
- Exercise 02: Superpowers Fresh-Agent Diff Triage - Run a fresh-agent review, triage blockers, and fix the highest-risk findings.
- Exercise 03: Promptfoo Review Regression Lab - Use Promptfoo to evaluate and improve code review behavior against regression cases.
- Exercise 04: Gitleaks PR Diff Review - Use Gitleaks and surrounding-code review to find and fix PR diff risk.
- Exercise 05: Graphiti Code Review Graph Skill - Use graph context to review and fix risk in a cross-cutting discount change.

## 10. Token Economics

Expensive context and reasoning are reserved for judgment-heavy work, while deterministic work moves to scripts, hooks, or smaller models.

- Exercise 01: Context7 Token-Budget Refactor - Use Context7 selectively while completing a budgeted refactor slice.
- Exercise 02: Context Cache Hygiene Challenge - Clean stale context cache material and prove the leaner setup with one change.
- Exercise 03: OpenAI Evals Model Routing Planner - Use eval evidence to create a model and effort routing policy.
- Exercise 04: Ponytail Minimal-Diff Budget - Use Ponytail-style minimal-diff discipline on a design-system migration slice.

## 11. Agentic Refactoring

Behavior is characterized before structure changes, and each refactor step leaves the suite green.

- Exercise 01: Characterization Test Refactor - Protect behavior with characterization tests before refactoring a rules module.
- Exercise 02: jscodeshift Strangler Checkout - Use a strangler slice and jscodeshift where useful to untangle checkout behavior.
- Exercise 03: OpenRewrite Legacy Rules Untangle - Use recipe-driven refactoring discipline to untangle a legacy rules boundary safely.

## 12. Agentic Retrospective

Repeated mistakes become rule, context, skill, or hook improvements, and waste trends down over time.

- Exercise 01: Session Waste Retro From Logs - Analyze session logs, reduce one repeated waste pattern, and verify the improvement.
- Exercise 02: Rule Hardening From Repeated Mistakes - Convert repeated agent mistakes into durable rules and prove them with a small change.
- Exercise 03: Superpowers Skill Improvement Loop - Use a retrospective loop to improve a flawed skill and hook setup.
- Exercise 04: OpenAI Evals Skill Optimizer - Use trace-based evals to improve a skill and measure the result.
