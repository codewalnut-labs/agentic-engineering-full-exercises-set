# Hook Failures

This is a seeded senior-lab input for Skill and Hook Improvement Loop. It gives the learner concrete constraints to inspect, implement, test, and verify.

## Operating Context

Skill and hook improvement loop from failed sessions

## Concrete Inputs

- flawed skill
- hook failure
- eval case
- improved trigger

## Seeded Risks

- skill triggers for unrelated reviews
- hook blocks harmless read commands
- eval checks output length but not correctness

## Verification Expectations

- before/after eval
- hook policy tests
- trigger negative cases
- maintenance note

## Agent Workflow Constraint

The learner must use an agent to inspect and plan, but the final implementation, review, and verification remain owned by the senior engineer.
